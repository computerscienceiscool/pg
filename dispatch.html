The dispatch

What a pg-note/v0 node does the moment bytes land on the wire. This is spec section 4, in plain terms.

protocolpg-note/v0
pCIDbafkreib7m46qk4gu6bthar64l3ytdssruvt23jonap5fxawoj2ugbuy7im

A message, on the wire

slot 0 · the pCID (a fingerprint) slot 1 · the note {from, body}

Every message is a two-part envelope. The dispatcher cares about slot 0 first — it decides whether slot 1 is even worth reading.

The dispatch, step by step

  1. Decode the bytes
    Parse the incoming CBOR into a structure. If the bytes don't parse, the node can't read the message — it notes the attempt and stops.
  2. Check the shape
    A real envelope is a two-element array whose slot 0 is CBOR tag 42 wrapping a byte string. Anything else isn't this protocol's envelope, so it's set aside.
  3. Read slot 0
    The byte string inside tag 42 is a pCID — a content fingerprint. This is the only field routing depends on. The note in slot 1 has still not been looked at.
  4. Compare the fingerprint
    Does slot 0's pCID equal this protocol's own pCID — the frozen hash of pg-note-v0.md? It's a plain byte-for-byte match against a constant the node already holds. No lookup, no registry, no authority consulted.
    Equal → deliver
    Hand slot 1 to the note handler, and record the delivery as evidence.
    Different → ignore
    Record "unknown protocol" and do nothing else. The note is never read.

That's the entire dispatcher. It routes on the fingerprint, not the content.

The three things you'll see in the log

✓ deliveredslot 0 matched this protocol — the note was handed off and recorded.
✕ unknown protocolslot 0 carried a different pCID — quietly ignored.
✕ couldn't decodethe bytes weren't a readable envelope — set aside.

Why it's shaped this way

Fingerprint first.Slot 1 is never interpreted until slot 0 has earned it. A node acts only on contracts it recognizes.
Ignore, don't crash.An unrecognized protocol is a quiet non-event, not an error. Unknown or hostile traffic simply doesn't route.
Everything leaves a trace.Delivered or ignored, each outcome is recorded as evidence — an append-only record you can read back later.
No central authority.The decision is local: a byte comparison against a constant. Nobody is asked for permission; no registry says what's valid.

What the dispatch deliberately doesn't decide yet: who sent the note. The from field is just a label anyone could claim. Judging a sender — signatures, promises, broken-promise evidence — is the next layer, and it sits on top of exactly this routing step.
